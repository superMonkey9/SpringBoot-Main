## 1. HTML 基本结构

> ⬇ [下一章：2. 常用标签](#2-常用标签) | [返回目录](#目录)

> 👉 [点击跳转到实操 Demo 1：第一个 HTML 页面](前端实操Demo.md#demo-1第一个-html-页面)

### 1.1 什么是 HTML？

HTML（HyperText Markup Language，超文本标记语言）是网页的**骨架**。

**生活类比：**

如果把网页比作一栋房子：
- **HTML** = 房子的框架（墙、门、窗的位置）
- **CSS** = 装修（颜色、样式、布局）
- **JavaScript** = 电器（让房子有功能：灯亮、门开）

### 1.2 基本结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>页面标题</title>
</head>
<body>
    页面内容写在这里
</body>
</html>
```

**逐行解释：**

| 标签 | 作用 |
|------|------|
| `<!DOCTYPE html>` | 声明这是一个 HTML5 文档 |
| `<html>` | 整个网页的根标签 |
| `<head>` | 放页面的"配置信息"（标题、编码、CSS引用等） |
| `<meta charset="UTF-8">` | 设置字符编码为 UTF-8，防止中文乱码 ⭐ |
| `<title>` | 浏览器标签页上显示的标题 |
| `<body>` | 页面的"内容区"，用户看到的所有东西都在这里 |

## 2. 常用标签

> ⬆ [上一章：1. HTML 基本结构](#1-html-基本结构) | ⬇ [下一章：3. 表单标签](#3-表单标签重点) | [返回目录](#目录)

>
> 👉 [点击跳转到实操 Demo 2：常用标签示范](前端实操Demo.md#demo-2常用标签示范)

### 2.1 标题标签 h1~h6

```html
<h1>一级标题（最大）</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
```

- h1 一个页面**最多用1次**（SEO 规范）
- 标题标签是**块级元素**，独占一行

### 2.2 段落标签 p

```html
<p>这是一个段落。段落之间会自动有间距。</p>
<p>这是另一个段落。</p>
```

### 2.3 div 标签 ⭐⭐⭐

div 是最常用的**容器标签**，本身没有特殊含义，主要用于**布局分组**。

```html
<div class="header">
    <h1>网站标题</h1>
</div>
<div class="content">
    <p>正文内容</p>
</div>
```

**生活类比：** div 就像一个"收纳盒"，你把相关的标签放在一起，方便后续用 CSS 统一样式。

### 2.4 span 标签

span 是**行内容器**，用于包裹一小段文字，方便单独设置样式。

```html
<p>这是一段话，<span style="color: red;">这部分是红色的</span>。</p>
```

**div vs span 的区别：**

| | div | span |
|---|---|---|
| 类型 | 块级元素（独占一行） | 行内元素（不换行） |
| 用途 | 布局分区 | 包裹文字片段 |

### 2.5 链接标签 a

```html
<!-- 基本用法 -->
<a href="https://www.baidu.com">点击跳转到百度</a>

<!-- 在新标签页打开 -->
<a href="https://www.baidu.com" target="_blank">新窗口打开百度</a>

<!-- 锚点跳转（页面内跳转） -->
<a href="#section1">跳到第一节</a>
<div id="section1">第一节内容</div>
```

**常用属性：**

| 属性 | 作用 |
|------|------|
| `href` | 跳转的目标地址 ⭐ |
| `target="_blank"` | 在新标签页中打开 |

### 2.6 图片标签 img

```html
<img src="photo.jpg" alt="这是一张照片" width="300">
```

**常用属性：**

| 属性 | 作用 |
|------|------|
| `src` | 图片路径 ⭐ |
| `alt` | 图片加载失败时显示的文字 ⭐ |
| `width` / `height` | 设置宽高 |

**路径问题（重要）：**

```html
<!-- 同级目录 -->
<img src="photo.jpg">

<!-- 子目录 -->
<img src="images/photo.jpg">

<!-- 上级目录 -->
<img src="../photo.jpg">
```

### 2.7 按钮标签 button

```html
<button>普通按钮</button>
<button type="button">点击按钮</button>
<button type="submit">提交按钮</button>
<button disabled>禁用按钮</button>
```

## 3. 表单标签（重点🔥）

> ⬆ [上一章：2. 常用标签](#2-常用标签) | ⬇ [下一章：4. 表格标签](#4-表格标签) | [返回目录](#目录)
> 📖 官方文档：[MDN 表单指南](https://developer.mozilla.org/zh-CN/docs/Learn/Forms)
>
> 👉 [点击跳转到实操 Demo 3：登录页面](前端实操Demo.md#demo-3登录页面)
> 👉 [点击跳转到实操 Demo 4：注册页面](前端实操Demo.md#demo-4注册页面)

> 表单是前端最重要的部分之一！登录、注册、搜索、提交数据全靠它！

### 3.1 form 标签

```html
<form action="/user/login" method="post">
    <!-- 表单内容 -->
    <input type="text" name="username">
    <input type="password" name="password">
    <button type="submit">登录</button>
</form>
```

**核心属性：**

| 属性 | 作用 |
|------|------|
| `action` | 提交到哪个地址 |
| `method` | 提交方式：GET 或 POST |

### 3.2 input 标签 ⭐⭐⭐

input 是表单中最常用的标签，通过 `type` 属性改变类型：

```html
<!-- 文本输入 -->
<input type="text" placeholder="请输入用户名">

<!-- 密码输入 -->
<input type="password" placeholder="请输入密码">

<!-- 邮箱输入 -->
<input type="email" placeholder="请输入邮箱">

<!-- 数字输入 -->
<input type="number" placeholder="请输入年龄">

<!-- 单选按钮（name 相同才能互斥） -->
<input type="radio" name="gender" value="male"> 男
<input type="radio" name="gender" value="female"> 女

<!-- 复选框 -->
<input type="checkbox" name="hobby" value="java"> Java
<input type="checkbox" name="hobby" value="python"> Python

<!-- 日期选择 -->
<input type="date">

<!-- 文件上传 -->
<input type="file">
```

> ⭐ **type 属性三大讲究！**

`type` 不只是换个样式那么简单，它决定了三件大事：

**讲究一：自动格式校验**
```html
<!-- type="text" — 啥都收，随便填 -->
<input type="text" value="乱写一通">    <!-- ✅ 通过 -->

<!-- type="email" — 必须有 @ 和域名 -->
<input type="email" value="乱写一通">   <!-- ❌ 浏览器报错 -->
<input type="email" value="abc@qq.com"> <!-- ✅ 通过 -->
```
不用写一行 JavaScript，浏览器**自动帮你检查**！

**讲究二：手机键盘不同**
- `type="text"` → 弹出普通键盘
- `type="email"` → 弹出带 `@` 和 `.com` 按钮的键盘
- `type="number"` → 弹出纯数字键盘
- `type="tel"` → 弹出拨号键盘

在手机上体验直接拉满！

**讲究三：后端也能受益**
`type` 越精确，前端过滤越严格，后端收到的数据格式就越规范，减少脏数据。

**常用 type 全家福：**

| type | 用途 | 浏览器行为 | 手机键盘 |
|------|------|-----------|---------|
| `text` | 普通文本 | 无校验 | 普通键盘 |
| `password` | 密码 | 输入内容显示为 `•••` | 普通键盘 |
| `email` | 邮箱 | 必须含 `@`，否则红框报错 | 带 `@` `.com` |
| `number` | 数字 | 只能输入数字，可能有上下箭头 | 纯数字键盘 |
| `tel` | 电话 | 无校验（格式太多样） | 拨号键盘 |
| `date` | 日期 | 弹出日期选择器 | 日期选择器 |
| `radio` | 单选 | 多个同 `name` 只能选一个 | — |
| `checkbox` | 多选 | 可以勾选多个 | — |
| `file` | 文件上传 | 弹出文件选择框 | 文件管理器 |
| `hidden` | 隐藏 | 用户看不到，但数据会提交 | — |
| `range` | 滑动条 | 显示一个滑块 | — |
| `color` | 颜色 | 弹出颜色选择器 | — |
| `submit` | 提交按钮 | 点击后提交表单 | — |

> 💡 **id 和 name 的区别（重要！）**
>
> 你在注册页面会看到 `id="username" name="username"`，虽然长得一样，但作用完全不同：
>
> | 属性 | 给谁用 | 作用 | 类比 |
> |------|--------|------|------|
> | `id` | 🖥️ 前端（浏览器/JS） | 唯一标识，用于 CSS 选择、JS 获取元素 | 快递单编号（仓库内部找包裹用） |
> | `name` | 📦 后端（服务器） | 表单提交时的 key，告诉服务器这数据叫啥 | 收件人姓名（收件人看这个） |
>
> ```javascript
> // id 给前端用
> let input = document.getElementById("username");
> ```
> ```java
> // name 给后端用（Spring Boot 参数名要和 name 一致）
> @PostMapping("/register")
> public String register(String username) { ... }
> ```
>
> 写成一样纯粹是因为**好记、方便**，完全可以不同：`id="user-input" name="username"`。

**核心属性：**

| 属性 | 作用 |
|------|------|
| `type` | 输入框类型 ⭐ |
| `name` | 表单字段名，提交时用的 key ⭐ |
| `placeholder` | 输入框的提示文字 |
| `value` | 默认值 |
| `required` | 必填项 |
| `disabled` | 禁用 |

> ⭐ **name 属性超级重要！** 没有 name 属性，数据就提交不了！就像快递单上的"收件人姓名"，没有它，数据不知道该写到哪。

### 3.5 label 标签（表单搭档⭐⭐）

> 📖 官方文档：[MDN label](https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/label)

> label 让表单更**好用**！点击文字就能选中对应的输入框。

```html
<!-- 基本用法：用 for 属性关联 input 的 id -->
<label for="username">用户名：</label>
<input type="text" id="username" name="username">

<!-- 简写：把 input 包在里面（省略 for） -->
<label>
    邮箱：
    <input type="email" name="email">
</label>
```

> ⭐ **为什么 label 超级重要？**
>
> 1. **点击文字 = 选中输入框** — 特别是 radio 和 checkbox，点击范围变大，用户不累
> 2. **无障碍访问** — 屏幕阅读器靠 label 知道每个输入框是干嘛的
> 3. **表单体验提升** — 专业网站全都在用

```html
<!-- 没有 label：用户必须精确点中小圆圈 ☹️ -->
<input type="radio" name="gender" value="male"> 男
<input type="radio" name="gender" value="female"> 女

<!-- 有 label：点击"男"或"女"文字也能选中 😊 -->
<input type="radio" name="gender" value="male" id="male">
<label for="male">男</label>
<input type="radio" name="gender" value="female" id="female">
<label for="female">女</label>
```

## 6. CSS 快速入门（白话版⭐⭐⭐）

> ⬆ [上一章：5. 语义化标签](#5-语义化标签与列表了解) | ⬇ [下一章：7. CSS 引入方式（详细版）](#7-css-引入方式) | [返回目录](#目录)
>
> 🎯 **本节是 CSS 的"从 0 到 1"导读！** 先看这节建立整体认知，再看后面的详细章节就不会迷路了~

### 6.1 CSS 到底在干嘛？

> **HTML = 骨架（是什么）**，**CSS = 外表（长什么样）**

**生活类比 🏠：**

| 角色 | 对应 | 干嘛的 |
|------|------|--------|
| **HTML** | 家具本体 | 有一张桌子、一把椅子、一个柜子 |
| **CSS** | 装修风格 | 桌子是白色的、椅子放在窗边、柜子高2米 |

- **没有 CSS 的网页** = 毛坯房（东西都在，但丑）
- **加上 CSS 的网页** = 精装修（好看、整齐、有设计感）

### 6.2 三种写法速览

**写法一：行内样式 — 直接贴在标签上（临时用）**
```html
<p style="color: red; font-size: 20px;">这段话是红色的</p>
```
> 类比：直接在衣服上贴标签 — "这件红色"。临时调试可以，正式项目别用。

**写法二：内部样式 — 写在 `<style>` 标签里（单页面）**
```html
<head>
    <style>
        p {
            color: red;
            font-size: 20px;
        }
    </style>
</head>
```
> 类比：在房间门口贴纸 — "这个房间里的桌子都刷红色"。

**写法三：外部样式 — 单独 `.css` 文件（⭐正式项目全用这种）**
```html
<link rel="stylesheet" href="style.css">
```
> 类比：写一本《装修手册》，所有房间都按手册来。**推荐！**

### 6.3 CSS 语法拆解（最核心！）

每一条 CSS 都长这样：

```css
选择器 {
    属性: 值;
    属性: 值;
}
```

真实例子：

```css
p {                   ← 选择器：选中所有 <p> 标签
    color: red;       ← 属性: 值;（文字颜色 = 红色）
    font-size: 16px;  ← 属性: 值;（字号 = 16像素）
}
```

⭐ **三个关键点：**
- 选择器后面跟 `{ }`（大括号）
- 每条样式写 `属性: 值;`（冒号分隔，分号结尾）
- 多条样式之间用 `;` 分号隔开

### 6.4 选择器白话版

选择器 = **告诉 CSS "你要改谁"**

| 选择器 | 写法 | 选几个 | 生活类比 |
|--------|------|--------|----------|
| 元素选择器 | `p { }` | 所有同名标签 | 老师说"所有男生站起来" → 全站 |
| class 选择器 | `.名字 { }` | 可以多个 | 老师说"戴红帽子的站起来" → 部分站 |
| id 选择器 | `#名字 { }` | 只能1个 | 老师说"学号001站起来" → 一个人站 |

```html
<!-- 实战例子 -->
<p>普通段落</p>                         <!-- 元素选择器 p{} 能选中 -->
<p class="error">错误信息</p>           <!-- class选择器 .error{} 能选中 -->
<h1 id="title">标题</h1>               <!-- id选择器 #title{} 能选中 -->
```

### 6.5 常用属性速查表

**文字相关：**
```css
.box {
    color: red;                /* 文字颜色 */
    font-size: 16px;          /* 字号 */
    font-weight: bold;        /* 加粗 */
    text-align: center;       /* 居中对齐 */
    text-decoration: none;    /* 去掉下划线（常用于链接） */
}
```

**背景相关：**
```css
.box {
    background-color: #f5f5f5;     /* 背景色 */
}
```

**尺寸相关：**
```css
.box {
    width: 200px;      /* 宽度 */
    height: 100px;     /* 高度 */
}
```

**显示模式：**
```css
.box {
    display: block;        /* 块级（独占一行，如 div） */
    display: inline;       /* 行内（不换行，如 span） */
    display: none;         /* 隐藏（不占位置） */
    display: flex;         /* 弹性布局（后面重点讲） */
}
```

### 6.6 盒模型白话版

> **每个 HTML 元素都是一个盒子**，这个盒子由 4 层组成。

**快递箱类比 📦：**
```
┌───────────────────────────────┐
│        margin（外边距）         │  ← 箱子和箱子之间的距离
│  ┌─────────────────────────┐  │
│  │      border（边框）       │  │  ← 箱子的纸板
│  │  ┌───────────────────┐  │  │
│  │  │  padding（内边距）  │  │  │  ← 箱子里的泡沫填充
│  │  │  ┌─────────────┐  │  │  │
│  │  │  │  content     │  │  │  │  ← 箱子里装的东西
│  │  │  └─────────────┘  │  │  │
│  │  └───────────────────┘  │  │
│  └─────────────────────────┘  │
└───────────────────────────────┘
```

```css
.box {
    width: 200px;           /* 内容宽度 */
    height: 100px;          /* 内容高度 */
    padding: 20px;          /* 内边距：内容到边框的距离 */
    border: 2px solid #333; /* 边框：2像素 实线 深灰色 */
    margin: 10px;           /* 外边距：盒子与盒子之间的距离 */
}
```

⭐ **最重要的坑：box-sizing**
```css
/* 不加这行（默认）：width:200 + padding:20×2 + border:2×2 = 实际宽度 244px 😱 */

/* 加了这行：width 就是最终宽度 ✅ */
* {
    box-sizing: border-box;
}
```
> ⭐ **每个项目第一行都写 `* { box-sizing: border-box; }`，省去 90% 的盒模型烦恼！**

### 6.7 Flex 布局白话版

> Flex = 让元素**排列整齐**的神器。开启方式：`display: flex;`

**两个轴要搞清楚：**
```
主轴（水平 →）：justify-content 控制
交叉轴（垂直 ↓）：align-items 控制
```

**最常用的 5 个组合：**
```css
/* 1. 水平+垂直居中（万能居中）⭐⭐⭐ */
.center {
    display: flex;
    justify-content: center;
    align-items: center;
}

/* 2. 两端对齐（导航栏常用）⭐⭐ */
.nav {
    display: flex;
    justify-content: space-between;
    align-items: center;
}

/* 3. 均匀分布（卡片列表）⭐⭐ */
.cards {
    display: flex;
    justify-content: space-around;
}

/* 4. 垂直排列 */
.column {
    display: flex;
    flex-direction: column;
}

/* 5. 子元素平均分配空间 */
.items {
    display: flex;
}
.item {
    flex: 1;  /* 每个 item 平分 */
}
```

>
> | 选择器 | 写法 | 选几个 | 生活类比 |

> ⭐ **实战口诀：** 全改用元素（`p{}`），部分改用 class（`.error{}`），唯一用 id（`#title{}`）。**90% 的情况用 class！**

### 7.5 伪类选择器（交互样式⭐⭐⭐）

> 📖 官方文档：[MDN 伪类](https://developer.mozilla.org/zh-CN/docs/Web/CSS/Pseudo-classes)

> 伪类 = 选择元素的**特定状态**，不用加 class，浏览器自动识别！

#### :hover 鼠标悬停 ⭐⭐⭐

```css
/* 鼠标移入按钮时变色 */
.btn:hover {
    background-color: #0056b3;
    cursor: pointer;        /* 鼠标变成小手 */
}

/* 鼠标移入链接时变色 */
a:hover {
    color: red;
    text-decoration: underline;
}
```

#### :focus 输入框聚焦 ⭐⭐⭐

```css
/* 用户点击输入框时，边框高亮 */
input:focus {
    border-color: #007bff;
    outline: none;           /* 去掉默认的蓝色轮廓 */
    box-shadow: 0 0 5px rgba(0, 123, 255, 0.3);  /* 发光效果 */
}
```

> 💡 `:focus` 对表单体验非常重要！用户能看到自己正在输入哪个框。

#### :nth-child 结构伪类（表格神器⭐⭐）

```css
/* 奇偶行交替颜色（斑马纹表格） */
tr:nth-child(even) {
    background-color: #f5f5f5;    /* 偶数行灰色 */
}

/* 第一个元素 */
li:first-child {
    font-weight: bold;
}

/* 最后一个元素 */
li:last-child {
    border-bottom: none;          /* 最后一行去掉边框 */
}

/* 第3个元素 */
li:nth-child(3) {
    color: red;
}
```

**生活类比：** `:nth-child` 就像老师点名 — "第偶数个同学站起来"、"第一个同学举手"。

## 9. 盒模型（核心🔥）

> ⬆ [上一章：8. 选择器](#8-选择器) | ⬇ [下一章：10. Flex 布局](#10-flex-布局核心) | [返回目录](#目录)
> 📖 官方文档：[MDN 盒模型](https://developer.mozilla.org/zh-CN/docs/Learn/CSS/Building_blocks/The_box_model)
>
> 👉 [点击跳转到实操 Demo 7：盒模型练习](前端实操Demo.md#demo-7盒模型练习)

> **这是 CSS 最重要的概念！** 每个 HTML 元素都是一个"盒子"。
>
> 💡 **白话速记：** 就像快递箱 —
> - **content** = 箱子里装的东西（你的文字/图片）
> - **padding** = 泡沫填充物（内容到箱壁的距离）
> - **border** = 箱子的纸板（边框本身）
> - **margin** = 箱子和箱子之间的距离（外间距）
>
> ⭐ **必写第一行：** `* { box-sizing: border-box; }` — 让 width 就是最终宽度，不会因为加 padding 变大！
>
> 👉 不加 box-sizing 的坑：设 width:200px，加 padding:20px，实际变成 240px，布局直接乱掉！

```
┌─────────────────────────────────┐
│            margin（外边距）        │
│  ┌───────────────────────────┐  │
│  │       border（边框）        │  │
│  │  ┌─────────────────────┐  │  │
│  │  │   padding（内边距）   │  │  │
│  │  │  ┌───────────────┐  │  │  │
│  │  │  │  content（内容）│  │  │  │
│  │  │  └───────────────┘  │  │  │
│  │  └─────────────────────┘  │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

```css
.box {
    width: 200px;
    height: 100px;
    padding: 20px;               /* 内边距：内容到边框的距离 */
    border: 1px solid #ccc;      /* 边框 */
    margin: 20px;                /* 外边距：盒子与盒子之间的距离 */
}
```

**生活类比：**
- **content** = 快递箱里的东西
- **padding** = 箱子里的泡沫填充物
- **border** = 箱子的纸板
- **margin** = 箱子和箱子之间的距离

### ⭐ box-sizing 重要！

```css
/* 推荐！让 width 包含 padding 和 border */
* {
    box-sizing: border-box;
}
```

> **不加 box-sizing 的坑：** 设 width: 200px，加 padding: 20px，实际宽度变成 240px。加了 `border-box` 后，实际宽度永远是 200px。

## 10. Flex 布局（核心🔥🔥🔥）

> ⬆ [上一章：9. 盒模型](#9-盒模型核心) | ⬇ [下一章：11. transition](#11-css-transition-过渡动画实用) | [返回目录](#目录)
> 📖 官方文档：[MDN Flex 布局](https://developer.mozilla.org/zh-CN/docs/Learn/CSS/CSS_layout/Flexbox)
> 📖 阮一峰：[Flex 布局教程](https://ruanyifeng.com/blog/2015/07/flex-grammar.html)
>
> 👉 [点击跳转到实操 Demo 8：Flex 布局实战](前端实操Demo.md#demo-flex-布局实战)

> 现代页面布局几乎全用 Flex！必须大量练习！

> 💡 **白话速记：** Flex 就是让子元素"排队"的工具。
> - 水平排队（默认）→ `flex-direction: row`
> - 垂直排队 → `flex-direction: column`
> - 居中 → `justify-content: center` + `align-items: center`
> - 两端对齐 → `justify-content: space-between`
> - 均匀分布 → `justify-content: space-around`
>
> ⭐ **万能居中公式：** `display: flex; justify-content: center; align-items: center;`
>
> 👉 详细的属性说明见下方，先记住上面这几句就够应付大部分场景了！

### 10.1 开启 Flex

```css
.container {
    display: flex;
}
```

> 加了 `display: flex` 的元素变成"弹性容器"，它的**子元素**自动变成"弹性项目"

### 10.2 主轴方向 flex-direction

```css
.container {
    flex-direction: row;            /* 水平排列（默认）→ */
    flex-direction: column;         /* 垂直排列 ↓ */
}
```

### 10.3 主轴对齐 justify-content ⭐⭐⭐

```css
.container {
    justify-content: flex-start;    /* 靠左（默认） */
    justify-content: center;        /* 居中 ⭐ */
    justify-content: space-between; /* 两端对齐 ⭐⭐ */
    justify-content: space-around;  /* 均匀分布（带半边距） */
    justify-content: space-evenly;  /* 完全均匀分布 */
}
```

### 10.4 交叉轴对齐 align-items ⭐⭐⭐

```css
.container {
    align-items: flex-start;    /* 靠上 */
    align-items: center;        /* 居中 ⭐ */
    align-items: stretch;       /* 拉伸填满（默认） */
}
```

### ⭐ 9.5 水平垂直居中万能写法

```css
.container {
    display: flex;
    justify-content: center;    /* 水平居中 */
    align-items: center;        /* 垂直居中 */
}
```

### 10.6 flex-wrap 换行

```css
.container {
    flex-wrap: wrap;            /* 允许换行 */
}
```

### 10.7 子元素属性

```css
.item {
    flex: 1;                    /* 占据剩余空间 */
    flex: 0 0 200px;           /* 固定宽度200px，不伸缩 */
}
```

## 11. CSS transition 过渡动画（实用🔥🔥）

> ⬆ [上一章：10. Flex 布局](#10-flex-布局核心) | ⬇ [下一章：12. 响应式设计](#12-响应式设计-media-媒体查询实用) | [返回目录](#目录)
> 📖 官方文档：[MDN transition](https://developer.mozilla.org/zh-CN/docs/Web/CSS/transition)

> 让状态变化（如 hover）变得**丝滑**，而不是瞬间切换！

```css
/* 基本语法：transition: 属性 名称 时间 */
.btn {
    background-color: blue;
    color: white;
    transition: background-color 0.3s ease;    /* 0.3秒平滑过渡 */
}
.btn:hover {
    background-color: darkblue;    /* 鼠标移入时，颜色 0.3 秒内渐变 */
}

/* 常见用法 */
.link {
    color: #333;
    transition: color 0.2s;        /* 文字颜色渐变 */
}
.link:hover {
    color: #007bff;
}

.card {
    transform: scale(1);
    transition: transform 0.3s;    /* 缩放渐变 */
}
.card:hover {
    transform: scale(1.05);        /* 鼠标移入微微放大 */
}
```

> 💡 **小技巧：** `transition: all 0.3s` 可以让所有属性都有过渡效果，但性能不如指定具体属性。

## 12. 响应式设计 @media 媒体查询（实用🔥🔥）

> ⬆ [上一章：11. transition](#11-css-transition-过渡动画实用) | ⬇ [下一章：13. 定位](#13-定位) | [返回目录](#目录)
> 📖 官方文档：[MDN 媒体查询](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_media_queries/Using_media_queries)

> 让页面在手机、平板、电脑上都能正常显示！

### 视口设置（必须加！）

```html
<!-- 必须放在 <head> 里，不加的话手机上页面会缩成蚂蚁大小 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

> ⭐ **不加这行，手机浏览器会按电脑宽度渲染页面，字小得看不见！**

### 媒体查询基本用法

```css
/* 默认样式（手机优先） */
.container {
    width: 100%;
    padding: 10px;
}

/* 平板（>= 768px） */
@media (min-width: 768px) {
    .container {
        width: 750px;
        margin: 0 auto;
    }
}

/* 电脑（>= 1024px） */
@media (min-width: 1024px) {
    .container {
        width: 960px;
    }
}
```

**生活类比：** 媒体查询就像"变色龙衣服"🦎 — 同一件衣服，在不同温度（屏幕大小）下自动变色（改变样式）。

### 实用场景：导航栏响应式

```css
/* 手机：导航栏竖排 */
.nav {
    display: flex;
    flex-direction: column;
}

/* 电脑：导航栏横排 */
@media (min-width: 768px) {
    .nav {
        flex-direction: row;
```

## 13. 定位

> ⬆ [上一章：12. 响应式设计](#12-响应式设计-media-媒体查询实用) | ⬇ [下一章：14. 基础语法](#14-基础语法) | [返回目录](#目录)

> 👉 [点击跳转到实操 Demo 9：定位练习](前端实操Demo.md#demo-9定位练习)

### 13.1 position 属性

```css
/* 相对定位：相对于自己原来的位置偏移 */
.box {
    position: relative;
    top: 10px;
    left: 20px;
}

/* 绝对定位：相对于最近的已定位祖先元素 */
.box {
    position: absolute;
    top: 0;
    right: 0;
}

/* 固定定位：相对于浏览器窗口 */
.navbar {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
}

/* 粘性定位：滚动到一定位置变成固定 */
.header {
    position: sticky;
    top: 0;
}
```

**生活类比：**
- `relative` = 站在原地，向左走两步（还占着原来的位置）
- `absolute` = 脱离队伍，跑到最近的参考点站着（不占原来的位置）
- `fixed` = 粘在墙上不动（滚动页面也不动）

### 13.2 常见定位场景

**右上角关闭按钮：**
```css
.parent {
    position: relative;     /* 父元素设为参考点 */
}
.close-btn {
    position: absolute;     /* 子元素绝对定位 */
    top: 10px;
    right: 10px;
}
```

**导航栏固定顶部：**
```css
.navbar {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    z-index: 999;           /* 层级，数字越大越在前面 */
}
```

## 22. localStorage 本地存储（重要🔥🔥）

> ⬆ [上一章：21. JSON](#21-json) | ⬇ [下一章：23. 联调流程](#23-联调流程) | [返回目录](#目录)

> 📖 官方文档：[MDN localStorage](https://developer.mozilla.org/zh-CN/docs/Web/API/Window/localStorage)

> 前端存储用户的登录状态（如 token）、偏好设置等，**关闭浏览器后数据还在**！

### 22.1 基本用法

```javascript
// 存数据
localStorage.setItem('username', '张三');
localStorage.setItem('token', 'abc123xyz');

// 取数据
let name = localStorage.getItem('username');   // '张三'
let token = localStorage.getItem('token');     // 'abc123xyz'

// 删数据
localStorage.removeItem('token');

// 清空所有
localStorage.clear();
```

### 22.2 ⭐ 存储对象/数组（必须用 JSON！）

```javascript
// 存对象 → 必须先转 JSON 字符串
let user = { name: '张三', age: 18, role: 'admin' };
localStorage.setItem('user', JSON.stringify(user));

// 取对象 → 必须再转回 JS 对象
let savedUser = JSON.parse(localStorage.getItem('user'));
console.log(savedUser.name);  // '张三'
```

> ⭐ **记忆口诀：** 存的时候 `stringify`，取的时候 `parse`，和 fetch 发送 JSON 一模一样！

### 22.3 实战场景：登录后保存 token

```javascript
// 登录成功后保存 token
const login = async () => {
    const response = await fetch('/user/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ username: 'zhangsan', password: '123456' })
    });
    const result = await response.json();
    if (result.code === 200) {
        // ⭐ 保存 token
        localStorage.setItem('token', result.data.token);
        localStorage.setItem('user', JSON.stringify(result.data.user));
        alert('登录成功！');
    }
};

// 发请求时带上 token
const loadUsers = async () => {
    const token = localStorage.getItem('token');
    const response = await fetch('/user/list', {
        headers: {
            'Authorization': token    // ⭐ 把 token 放到请求头里
        }
    });
    const result = await response.json();
    console.log(result.data);
};

// 退出登录
const logout = () => {
    localStorage.removeItem('token');
    localStorage.removeItem('user');
    window.location.href = '/login.html';
};
```

> **localStorage vs sessionStorage：**
>
> | | localStorage | sessionStorage |
> |---|---|---|
> | 生命周期 | 永久，除非手动删除 | 关闭标签页就没了 |
> | 使用频率 | ⭐⭐⭐ 很常用 | 偶尔用 |

# 附录：前端关键词速查表

| 关键词 | 含义 |
|--------|------|
| **HTML** | 超文本标记语言，网页的骨架 |
| **CSS** | 层叠样式表，网页的外观 |
| **JavaScript** | 网页的编程语言，网页的行为 |
| **DOM** | 文档对象模型，JS 操作 HTML 的接口 |
| **Flex** | 弹性布局，现代 CSS 布局的核心 |
| **盒模型** | CSS 中每个元素都是一个盒子（content+padding+border+margin） |
| **选择器** | CSS 中选中元素的方式（class、id、后代等） |
| **fetch** | 浏览器内置的网络请求 API |
| **JSON** | 前后端数据交换的文本格式 |
| **async/await** | 异步编程语法，让异步代码看起来像同步 |
| **Promise** | 异步操作的容器 |
| **addEventListener** | 给元素绑定事件的方法 |
| **querySelector** | 通过 CSS 选择器获取 DOM 元素 |
| **textContent** | 获取/设置元素的文字内容 |
| **innerHTML** | 获取/设置元素的 HTML 内容 |
| **classList** | 操作元素的 class 属性 |
| **e.preventDefault()** | 阻止表单默认提交行为 |
| **JSON.stringify()** | JS 对象 → JSON 字符串 |
| **JSON.parse()** | JSON 字符串 → JS 对象 |
| **Content-Type** | HTTP 请求头，告诉后端发送的数据格式 |
| **跨域** | 前端和后端不在同一个域名/端口时的限制 |
| **XSS** | 跨站脚本攻击，用 innerHTML 时需要注意 |
