## 1. HTML 基本结构

> ⬇ [下一章：2. 常用标签](#2-常用标签) | [返回目录](#目录)

>
> 👉 [点击跳转到实操 Demo 1：第一个 HTML 页面](前端实操Demo.md#demo-1第一个-html-页面)

### 1.1 什么是 HTML？

HTML（HyperText Markup Language，超文本标记语言）是网页的**骨架**。

**生活类比：**

如果把网页比作一栋房子：
- **HTML** = 房子的框架（墙、门、窗的位置）
- **CSS** = 装修（颜色、样式、布局）
- **JavaScript** = 电器（让房子有功能：灯亮、门开）

### 1.2 基本结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>页面标题</title>
</head>
<body>
    页面内容写在这里
</body>
</html>
```

**逐行解释：**

| 标签 | 作用 |
|------|------|
| `<!DOCTYPE html>` | 声明这是一个 HTML5 文档 |
| `<html>` | 整个网页的根标签 |
| `<head>` | 放页面的"配置信息"（标题、编码、CSS引用等） |
| `<meta charset="UTF-8">` | 设置字符编码为 UTF-8，防止中文乱码 ⭐ |
| `<title>` | 浏览器标签页上显示的标题 |
| `<body>` | 页面的"内容区"，用户看到的所有东西都在这里 |

## 2. 常用标签

> ⬆ [上一章：1. HTML 基本结构](#1-html-基本结构) | ⬇ [下一章：3. 表单标签](#3-表单标签重点) | [返回目录](#目录)

>
> 👉 [点击跳转到实操 Demo 2：常用标签示范](前端实操Demo.md#demo-2常用标签示范)

### 2.1 标题标签 h1~h6

```html
<h1>一级标题（最大）</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
```

- h1 一个页面**最多用1次**（SEO 规范）
- 标题标签是**块级元素**，独占一行

### 2.2 段落标签 p

```html
<p>这是一个段落。段落之间会自动有间距。</p>
<p>这是另一个段落。</p>
```

### 2.3 div 标签 ⭐⭐⭐

div 是最常用的**容器标签**，本身没有特殊含义，主要用于**布局分组**。

```html
<div class="header">
    <h1>网站标题</h1>
</div>
<div class="content">
    <p>正文内容</p>
</div>
```

**生活类比：** div 就像一个"收纳盒"，你把相关的标签放在一起，方便后续用 CSS 统一样式。

### 2.4 span 标签

span 是**行内容器**，用于包裹一小段文字，方便单独设置样式。

```html
<p>这是一段话，<span style="color: red;">这部分是红色的</span>。</p>
```

**div vs span 的区别：**

| | div | span |
|---|---|---|
| 类型 | 块级元素（独占一行） | 行内元素（不换行） |
| 用途 | 布局分区 | 包裹文字片段 |

### 2.5 链接标签 a

```html
<!-- 基本用法 -->
<a href="https://www.baidu.com">点击跳转到百度</a>

<!-- 在新标签页打开 -->
<a href="https://www.baidu.com" target="_blank">新窗口打开百度</a>

<!-- 锚点跳转（页面内跳转） -->
<a href="#section1">跳到第一节</a>
<div id="section1">第一节内容</div>
```

**常用属性：**

| 属性 | 作用 |
|------|------|
| `href` | 跳转的目标地址 ⭐ |
| `target="_blank"` | 在新标签页中打开 |

### 2.6 图片标签 img

```html
<img src="photo.jpg" alt="这是一张照片" width="300">
```

**常用属性：**

| 属性 | 作用 |
|------|------|
| `src` | 图片路径 ⭐ |
| `alt` | 图片加载失败时显示的文字 ⭐ |
| `width` / `height` | 设置宽高 |

**路径问题（重要）：**

```html
<!-- 同级目录 -->
<img src="photo.jpg">

<!-- 子目录 -->
<img src="images/photo.jpg">

<!-- 上级目录 -->
<img src="../photo.jpg">
```

### 2.7 按钮标签 button

```html
<button>普通按钮</button>
<button type="button">点击按钮</button>
<button type="submit">提交按钮</button>
<button disabled>禁用按钮</button>
```

## 3. 表单标签（重点🔥）

> ⬆ [上一章：2. 常用标签](#2-常用标签) | ⬇ [下一章：4. 表格标签](#4-表格标签) | [返回目录](#目录)
> 📖 官方文档：[MDN 表单指南](https://developer.mozilla.org/zh-CN/docs/Learn/Forms)
>
> 👉 [点击跳转到实操 Demo 3：登录页面](前端实操Demo.md#demo-3登录页面)
> 👉 [点击跳转到实操 Demo 4：注册页面](前端实操Demo.md#demo-4注册页面)

> 表单是前端最重要的部分之一！登录、注册、搜索、提交数据全靠它！

### 3.1 form 标签

```html
<form action="/user/login" method="post">
    <!-- 表单内容 -->
    <input type="text" name="username">
    <input type="password" name="password">
    <button type="submit">登录</button>
</form>
```

**核心属性：**

| 属性 | 作用 |
|------|------|
| `action` | 提交到哪个地址 |
| `method` | 提交方式：GET 或 POST |

### 3.2 input 标签 ⭐⭐⭐

input 是表单中最常用的标签，通过 `type` 属性改变类型：

```html
<!-- 文本输入 -->
<input type="text" placeholder="请输入用户名">

<!-- 密码输入 -->
<input type="password" placeholder="请输入密码">

<!-- 邮箱输入 -->
<input type="email" placeholder="请输入邮箱">

<!-- 数字输入 -->
<input type="number" placeholder="请输入年龄">

<!-- 单选按钮（name 相同才能互斥） -->
<input type="radio" name="gender" value="male"> 男
<input type="radio" name="gender" value="female"> 女

<!-- 复选框 -->
<input type="checkbox" name="hobby" value="java"> Java
<input type="checkbox" name="hobby" value="python"> Python

<!-- 日期选择 -->
<input type="date">

<!-- 文件上传 -->
<input type="file">
```

> ⭐ **type 属性三大讲究！**

`type` 不只是换个样式那么简单，它决定了三件大事：

**讲究一：自动格式校验**
```html
<!-- type="text" — 啥都收，随便填 -->
<input type="text" value="乱写一通">    <!-- ✅ 通过 -->

<!-- type="email" — 必须有 @ 和域名 -->
<input type="email" value="乱写一通">   <!-- ❌ 浏览器报错 -->
<input type="email" value="abc@qq.com"> <!-- ✅ 通过 -->
```
不用写一行 JavaScript，浏览器**自动帮你检查**！

**讲究二：手机键盘不同**
- `type="text"` → 弹出普通键盘
- `type="email"` → 弹出带 `@` 和 `.com` 按钮的键盘
- `type="number"` → 弹出纯数字键盘
- `type="tel"` → 弹出拨号键盘

在手机上体验直接拉满！

**讲究三：后端也能受益**
`type` 越精确，前端过滤越严格，后端收到的数据格式就越规范，减少脏数据。

**常用 type 全家福：**

| type | 用途 | 浏览器行为 | 手机键盘 |
|------|------|-----------|---------|
| `text` | 普通文本 | 无校验 | 普通键盘 |
| `password` | 密码 | 输入内容显示为 `•••` | 普通键盘 |
| `email` | 邮箱 | 必须含 `@`，否则红框报错 | 带 `@` `.com` |
| `number` | 数字 | 只能输入数字，可能有上下箭头 | 纯数字键盘 |
| `tel` | 电话 | 无校验（格式太多样） | 拨号键盘 |
| `date` | 日期 | 弹出日期选择器 | 日期选择器 |
| `radio` | 单选 | 多个同 `name` 只能选一个 | — |
| `checkbox` | 多选 | 可以勾选多个 | — |
| `file` | 文件上传 | 弹出文件选择框 | 文件管理器 |
| `hidden` | 隐藏 | 用户看不到，但数据会提交 | — |
| `range` | 滑动条 | 显示一个滑块 | — |
| `color` | 颜色 | 弹出颜色选择器 | — |
| `submit` | 提交按钮 | 点击后提交表单 | — |

> 💡 **id 和 name 的区别（重要！）**
>
> 你在注册页面会看到 `id="username" name="username"`，虽然长得一样，但作用完全不同：
>
> | 属性 | 给谁用 | 作用 | 类比 |
> |------|--------|------|------|
> | `id` | 🖥️ 前端（浏览器/JS） | 唯一标识，用于 CSS 选择、JS 获取元素 | 快递单编号（仓库内部找包裹用） |
> | `name` | 📦 后端（服务器） | 表单提交时的 key，告诉服务器这数据叫啥 | 收件人姓名（收件人看这个） |
>
> ```javascript
> // id 给前端用
> let input = document.getElementById("username");
> ```
> ```java
> // name 给后端用（Spring Boot 参数名要和 name 一致）
> @PostMapping("/register")
> public String register(String username) { ... }
> ```
>
> 写成一样纯粹是因为**好记、方便**，完全可以不同：`id="user-input" name="username"`。

**核心属性：**

| 属性 | 作用 |
|------|------|
| `type` | 输入框类型 ⭐ |
| `name` | 表单字段名，提交时用的 key ⭐ |
| `placeholder` | 输入框的提示文字 |
| `value` | 默认值 |
| `required` | 必填项 |
| `disabled` | 禁用 |

> ⭐ **name 属性超级重要！** 没有 name 属性，数据就提交不了！就像快递单上的"收件人姓名"，没有它，数据不知道该写到哪。

## 6. CSS 引入方式

> ⬆ [上一章：5. 语义化标签](#5-语义化标签与列表了解) | ⬇ [下一章：7. 选择器](#7-选择器) | [返回目录](#目录)
> 📖 官方文档：[MDN CSS 入门](https://developer.mozilla.org/zh-CN/docs/Learn/CSS)
> 📖 W3School：[CSS 教程](https://www.w3school.com.cn/css/index.asp)

### 6.1 行内样式（直接写在标签上）

```html
<p style="color: red; font-size: 16px;">红色文字</p>
```

### 6.2 内部样式（写在 style 标签中）

```html
<head>
    <style>
        p {
            color: red;
            font-size: 16px;
        }
    </style>
</head>
```

### 6.3 外部样式（推荐⭐⭐⭐）

```html
<!-- HTML 中引入 -->
<link rel="stylesheet" href="style.css">
```

```css
/* style.css 文件 */
p {
    color: red;
    font-size: 16px;
}
```

**三种方式对比：**

| 方式 | 适用场景 | 推荐程度 |
|------|----------|----------|
| 行内样式 | 临时调试 | ❌ 不推荐 |
| 内部样式 | 单页面小项目 | ⭐ 一般 |
| 外部样式 | 正式项目 | ⭐⭐⭐ 强烈推荐 |

## 7. 选择器

> ⬆ [上一章：6. CSS 引入方式](#6-css-引入方式) | ⬇ [下一章：8. 盒模型](#8-盒模型核心) | [返回目录](#目录)
> 📖 官方文档：[MDN CSS 选择器](https://developer.mozilla.org/zh-CN/docs/Learn/CSS/Building_blocks/Selectors)
>
> 👉 [点击跳转到实操 Demo 6：CSS 选择器练习](前端实操Demo.md#demo-6css-选择器练习)

### 7.1 元素选择器

```css
/* 选中所有 p 标签 */
p {
    color: blue;
}
```

### 7.2 class 选择器 ⭐⭐⭐

```html
<p class="error-msg">错误信息</p>
```

```css
/* 以 . 开头 */
.error-msg {
    color: red;
    font-size: 14px;
}
```

> class 可以重复使用，一个元素可以有多个 class（空格分隔）

### 7.3 id 选择器

```html
<h1 id="main-title">主标题</h1>
```

```css
/* 以 # 开头 */
#main-title {
    color: #333;
}
```

> id 在一个页面中**必须唯一**，实际开发中用得比 class 少

### 7.4 后代选择器

```css
/* 选中 .header 内部所有的 p */
.header p {
    color: white;
}
```

### 7.5 选择器优先级

```
行内样式 > id选择器 > class选择器 > 元素选择器
```

> 👉 [点击跳转到实操 Demo 7：盒模型练习](前端实操Demo.md#demo-7盒模型练习)

> **这是 CSS 最重要的概念！** 每个 HTML 元素都是一个"盒子"。

```
┌─────────────────────────────────┐
│            margin（外边距）        │
│  ┌───────────────────────────┐  │
│  │       border（边框）        │  │
│  │  ┌─────────────────────┐  │  │
│  │  │   padding（内边距）   │  │  │
│  │  │  ┌───────────────┐  │  │  │
│  │  │  │  content（内容）│  │  │  │
│  │  │  └───────────────┘  │  │  │
│  │  └─────────────────────┘  │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

```css
.box {
    width: 200px;
    height: 100px;
    padding: 20px;               /* 内边距：内容到边框的距离 */
    border: 1px solid #ccc;      /* 边框 */
    margin: 20px;                /* 外边距：盒子与盒子之间的距离 */
}
```

**生活类比：**
- **content** = 快递箱里的东西
- **padding** = 箱子里的泡沫填充物
- **border** = 箱子的纸板
- **margin** = 箱子和箱子之间的距离

### ⭐ box-sizing 重要！

```css
/* 推荐！让 width 包含 padding 和 border */
* {
    box-sizing: border-box;
}
```

> **不加 box-sizing 的坑：** 设 width: 200px，加 padding: 20px，实际宽度变成 240px。加了 `border-box` 后，实际宽度永远是 200px。

## 10. 定位

> ⬆ [上一章：9. Flex 布局](#9-flex-布局核心) | ⬇ [下一章：11. 基础语法](#11-基础语法) | [返回目录](#目录)

>
> 👉 [点击跳转到实操 Demo 9：定位练习](前端实操Demo.md#demo-9定位练习)

### 10.1 position 属性

```css
/* 相对定位：相对于自己原来的位置偏移 */
.box {
    position: relative;
    top: 10px;
    left: 20px;
}

/* 绝对定位：相对于最近的已定位祖先元素 */
.box {
    position: absolute;
    top: 0;
    right: 0;
}

/* 固定定位：相对于浏览器窗口 */
.navbar {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
}

/* 粘性定位：滚动到一定位置变成固定 */
.header {
    position: sticky;
    top: 0;
}
```

**生活类比：**
- `relative` = 站在原地，向左走两步（还占着原来的位置）
- `absolute` = 脱离队伍，跑到最近的参考点站着（不占原来的位置）
- `fixed` = 粘在墙上不动（滚动页面也不动）

### 10.2 常见定位场景

**右上角关闭按钮：**
```css
.parent {
    position: relative;     /* 父元素设为参考点 */
}
.close-btn {
    position: absolute;     /* 子元素绝对定位 */
    top: 10px;
    right: 10px;
}
```

**导航栏固定顶部：**
```css
.navbar {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    z-index: 999;           /* 层级，数字越大越在前面 */
}
```

JSON（JavaScript Object Notation）是前后端数据交换的**通用格式**。

```javascript

console.log('响应状态:', response.status);
const result = await response.json();
console.log('返回数据:', result);
```

# 附录：前端关键词速查表

| 关键词 | 含义 |
|--------|------|
| **HTML** | 超文本标记语言，网页的骨架 |
| **CSS** | 层叠样式表，网页的外观 |
| **JavaScript** | 网页的编程语言，网页的行为 |
| **DOM** | 文档对象模型，JS 操作 HTML 的接口 |
| **Flex** | 弹性布局，现代 CSS 布局的核心 |
| **盒模型** | CSS 中每个元素都是一个盒子（content+padding+border+margin） |
| **选择器** | CSS 中选中元素的方式（class、id、后代等） |
| **fetch** | 浏览器内置的网络请求 API |
| **JSON** | 前后端数据交换的文本格式 |
| **async/await** | 异步编程语法，让异步代码看起来像同步 |
| **Promise** | 异步操作的容器 |
| **addEventListener** | 给元素绑定事件的方法 |
| **querySelector** | 通过 CSS 选择器获取 DOM 元素 |
| **textContent** | 获取/设置元素的文字内容 |
| **innerHTML** | 获取/设置元素的 HTML 内容 |
| **classList** | 操作元素的 class 属性 |
| **e.preventDefault()** | 阻止表单默认提交行为 |
| **JSON.stringify()** | JS 对象 → JSON 字符串 |
| **JSON.parse()** | JSON 字符串 → JS 对象 |
| **Content-Type** | HTTP 请求头，告诉后端发送的数据格式 |
| **跨域** | 前端和后端不在同一个域名/端口时的限制 |
| **XSS** | 跨站脚本攻击，用 innerHTML 时需要注意 |
